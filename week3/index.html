This

What is 'this'? This is a keyword in JS that refers to an object. The way this keyword is used will depend on how we want to call or use it. When we have methods we can use "this" to access the object in a method. Other characteristics "this" is that it can be used in any function. When the keyword "this" is alone, or in a function it refers to a global object. When it is in an event, "this" refers to the element that received the event.When used in an object method, "this" refers to the object.

Example:
let carModel = {
color: "Red",
year: 2004,
brand: BMW,
showCar() {
// "this" is the "current object"
console.log(this.brand);
}
};
carModel.showCar();

The value of this is defined at run-time.

      When a function is declared, it may use this, but that this has no value until the function is called.
      A function can be copied between objects.
      When a function is called in the “method” syntax: object.method(), the value of this during the call is object.

    A variable is called a property and function is called a method inside an object.

    Inside an object we create properties and values to it.

Objects

An object in JavaScript is a self-contained set of related values and functions. They act as a collection of named properties that map to any JavaScript value such as strings, numbers, booleans, arrays and functions. If a property’s value is a function, it is known as a method.One way to think about an object is that it’s like a dictionary where you look up a property name and see a value.Objects are often used to keep any related information and functionality together in the same place.

Object Literal
An object literal is an object that is created directly in the language by wrapping all its properties and methods in curly braces {}. Object literals are a distinguishing feature of the JavaScript language, as they allow objects to be created quickly without the need for defining a class. Each property inside an onject is a key-value pair, separated by commas. All objects are mutable at any time when a program is running. This means that its properties and methods can be changed or removed, and new properties and methods can be added to the object, even if it was declared using const.

To create a new object we just need to add curly braces {} to a declared variable.

Example
const car = {};

ES6 provided a shorthand method of creating objects if a property key is the same as a variable name that the property value is assigned to:

Example
const name = 'Iron Man';
const realName = 'Tony Stark';
const ironMan = { name, realName };

hasOwnProperty(). This can be used to check whether an object has a property that is its own, rather than one that has been inherited from another object:

true

hasOwnProperty() method will only return any properties that belong to that particular object, whereas using in or !== undefined will return true, even if the property has been inherited from another object.

Document Object Model

The Document Object Model, or DOM for short, represents an HTML document as a network of connected nodes that form a tree-like structure.The DOM treats everything on a web page as a node. HTML tags

The Document Object Model (DOM) allows you to access elements of a web page and enable interaction with the page by adding and removing elements, changing the order, content and attributes of elements, and even altering how they are styled.The DOM is not from JavaScript

Getting Elements the DOM has different methods to access any element on a page.These methods will return a node object or a node list, which is an array-like object. These objects can then be assigned to a variable and be inspected or modified.

Node type objects have number of properties and methods that we can use to find information about, or modify, from an element.Nodes have a numerical code to signify what type they are

    Numeric codes for nodeTypes
    1 = element
    2 = attribute
    3 = text
    8 = comment
    9 = body
    DOM shortcut methods
    Document.body: returns the body element of a web page, as we saw in the previous example.
    Document.images: returns a node list of all the images contained in the document.
    Document.links: returns a node list of all the elements and elements that have an href attribute.
    Document.anchors: returns a node list of all the elements that have a name attribute.
    Document.forms: returns a node list of all the forms in the document.
    getElementById(): returns a reference to the element with a unique id attribute that is given as an argument.
    getElementsByTagName(): returns a live node list of all the elements with the tag name that is provided as an argument.
    getElementsByClassName(): returns a live node list of all elements that have the class name that is supplied as an argument.
    document.querySelector(): allows you to use CSS notation to find the first element in the document that matches that matches a CSS selector provided as an argument.
    document.querySelectorAll(): uses CSS notation but returns a node list of all the elements in the document that match the CSS query selector.
    querySelector(): can be called on any element, rather than just document

Node objects have a number of properties and methods for navigating around the document tree. Once you have a reference to an element, you can walk along the document tree to find other nodes.

Events

Events are also part of the DOM and they provides the link between the web page and user interactions. Every time a user interacts with a web page, such as clicking on a link, pressing a key, or moving a mouse, an event occurs that our program can detect and then respond to.

Event Listenersre like setting a notification to alert you when something happens. The event listener let you know when the event happens and the program can then respond appropriately. This allows the program to continue with other tasks while it waits for the event to happen.JavaScript, uses a non-blocking approach that uses event listeners to listen out for any clicks on the page. Every time the page is clicked, a callback function will be called.The click event occurs when a user clicks with the mouse, presses the Enter key, or taps the screen, making it a very useful all-round event covering many types of interaction. To call an event listener we use the addEventListener() method. This method is called on a node object, the node to which the event listener is being applied.

Example:
document.main.addEventListener("click", doSomething);

The addEventListener() method can also be called without a node, in which case it is applied to the global object, usually the whole browser window.

Types of Events

The type property returns the type of event that occurred

      target: the target property returns a reference to the node that fired the event
      screenX and screenY: show the number of pixels from the left and top of the screen respectively where the event took place.
      clientX and clientY: show the number of pixels from the left and top of the client that is being used (usually the browser window).
      pageX and pageY: show the number of pixels from the left and top, respectively, where the event took place in the document. This property takes account of whether the page has been scrolled.
      dblclick: when the user doubleclicks on the element to which the event listener is attached.

      mouse events: the click event that occurs when a mouse button is clicked.
      mouseover: when the mouse pointer is placed over the element to which the event listener is attached.
      mouseout: when the mouse pointer moves away from an element.
      mousemove: whenever the mouse moves.

      keydown:when a key is pressed and will continue to occur if the key is held down.
      keypress:occurs after a keydown event but before a keyup event. The keypress event only occurs for keys that produce character input (plus the 'Delete' key).It is use to to find out the character that was pressed on the keyboard.
      key up:when a key is released.

      modifier keys: pressing the modifier keys such as Shift, Ctrl, Alt and meta (Cmd on Mac) will fire the keydown and keyup events, but not the keypress event as they don't produce any characters on the screen.The name of the modifier key is still returned by the key property.The shiftKey, ctrlKey, altKey, and metaKey are all properties of the event object and return true if the relevant key was held down.

      touchstart: when a user initially touches the surface.They are used to support smarthphones or tablets.
      touchmove: occurs after a user has touched the screen then moves around without leaving.
      touchenter: occurs when a user has already started touching the surface.
      touchleave: occurs when the user is still touching the surface.
      touchcancel:occurs when a touch event is interrupted, such as a user’s finger moving outside the document window, or too many fingers being used at once.

Removing Event Listeners An event listener can be removed using the removeEventListener() method. This method makes the click lsitener to only be called once.
