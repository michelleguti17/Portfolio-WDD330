Testing and Debugging

Errors are caused when something goes wrong in a program. They are usually caused by: System error, Programmer error, User error.


Exceptions an exception is an error that produces a return value that can then be used by the program to deal with the error.

Stack trace this is a sequence of functions or method calls that lead to the point where the error occurred. An exception can also produce a stack trace.

Warnings a warning can occur if there’s an error in the code that isn't enough to cause the program to crash. This means the program will continue to run after a warning.


Debugging Methods

Linting Tools linting tools such as JS Lint,JS Hint, and ES Lint are designed to highlight any sloppy programming practices or syntax errors, and will complain if certain style conventions are not followed, such as how code is indented.Passing a lint test is no guarantee that your code is correct, but it will mean it will be more consistent and less likely to have problems.

Feature Detection is used to determine if a browser support a feature.This is done using an if statement to check whether an object or method exists before trying to actually call the method.

Alert the alert() method to show a dialog at certain points in the code.It allows us to effectively put breakpoints in the code that let us check the value of variables at that point to see if they’re what we expect them to be

Console.log() this can be used to log the value of variables at different stages of the program, although it will not actually stop the execution of the program in the same way as alert() does.

Console.trace() it shows an an interactive stack trace in the console. This will show the functions that were called in the lead up to an exception occurring while the code is running.

Debugger t his is a keyword that can be used in different browsers and will allow us to create a breakpoint in our code that will pause the execution of the code and allow us to see where the program is currently up to.

Exception Handling is the best way to catch errors and identify where the errors are generated despite the program does not crash or the users do not see the error.To use Exception Handling properly is necessary to use try,catch and finally.

class MultiplicatorUnitFailure extends Error


function primitiveMultiply(a, b) {
if (Math.random() < 0.2) {
return a * b;
} else {
throw new MultiplicatorUnitFailure("Klunk");
}
}


function reliableMultiply(a, b) {
for (;;) {
try {
return primitiveMultiply(a, b);
} catch (error) {
if (error instanceof MultiplicatorUnitFailure == false)
throw error;
}
}
}
}


console.log(reliableMultiply(8, 8));
// → 64
